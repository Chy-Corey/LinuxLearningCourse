## 用户管理

> Create Date：2026-04-03
>
> Update Date: 2026-04-03

与 Windows 不同，登录 Windows 系统时通常只需输入密码即可——这是因为一台运行 Windows（尤其是家用版）的计算机通常默认只供单个用户使用，系统设计也围绕这一场景展开。

而 Linux 则是一个**多用户、多任务的分时操作系统**，其核心设计理念从一开始就面向**多用户共享同一台计算机**的场景（如早期的服务器、工作站等）。在 Linux 中，多个用户可以同时通过本地终端、SSH等方式登录系统，并各自独立地运行程序、访问文件，彼此之间互不干扰。

正是由于 Linux 的多用户特性，系统必须具备**严格、精细的用户权限管理机制**。因此，Linux 从内核层面就引入了完善的权限控制体系——包括**用户（User）、用户组（Group）和权限位（Read/Write/Execute）的三元模型**，以及基于文件所有者、所属组和其他用户的访问控制规则。

因此，理解用户管理是掌握 Linux 基础操作的关键。

---

### 一、Linux 用户的类型以及用户管理

用户类型包括：

1. **root 用户（超级管理员）**
   - UID 为 0，拥有对系统的**完全控制权**
   - 可以修改任何文件、安装软件、管理服务等
   - **不建议日常使用 root**，容易误操作导致系统损坏
2. **普通用户**
   - 由管理员创建，用于日常操作
   - 默认只能访问自己的home目录（如 `/home/yubao`）和部分公共目录
   - 可通过 `sudo` 临时获得 root 权限（需授权）

管理员可以创建、删除、修改普通用户，命令方式如下：

| 命令                    | 说明                                                 |
| ----------------------- | ---------------------------------------------------- |
| `useradd yubao`         | 创建用户 `yubao`（不创建家目录）                     |
| `adduser yubao`         | 交互式创建用户（自动创建家目录、设置密码等，更友好） |
| `passwd yubao`          | 为用户 `yubao` 设置或修改密码                        |
| `usermod -aG dev yubao` | 将 `yubao` 添加到 `dev` 组（`-aG` 表示追加）         |
| `userdel yubao`         | 删除用户（加 `-r` 同时删除家目录）                   |

以上及以下的命令不需要刻意去背，在开发过程中记忆即可，命令有非常多的选项符，可参考链接[Linux 用户和用户组管理 | 菜鸟教程](https://www.runoob.com/linux/linux-user-manage.html)。

### 二、用户与用户组的关系

- 每个用户必须属于**至少一个主组**（Primary Group），默认组名与用户名相同
- 用户可以同时属于**多个附加组**（Supplementary Groups）
- 权限可按**用户**或**组**分配，便于批量管理

> 🌰 举例：将开发人员 `yubao` 和 `bob` 加入 `dev` 组，然后赋予 `dev` 组对 `/project` 目录的读写权限，两人即可协作。

| 命令           | 说明                      |
| -------------- | ------------------------- |
| `groupadd dev` | 创建 `dev` 用户组         |
| `groupdel dev` | 删除 `dev` 用户组         |
| `groups yubao` | 查看 `yubao` 所属的所有组 |

### 三、核心配置文件

| 文件           | 作用                                                     |
| -------------- | -------------------------------------------------------- |
| `/etc/passwd`  | 存储用户基本信息（用户名、UID、GID、家目录、默认 Shell） |
| `/etc/shadow`  | 存储加密后的密码（仅 root 可读）                         |
| `/etc/group`   | 存储用户组信息（组名、GID、成员列表）                    |
| `/etc/sudoers` | 配置哪些用户或组可以使用 `sudo`（用 `visudo` 编辑）      |

### 四、文件/目录权限

每个文件/目录有三类权限对象：
- **所有者（User）**
- **所属组（Group）**
- **其他人（Others）**

每类有三种权限：
- **r（读）**：查看文件内容 / 列出目录
- **w（写）**：修改文件 / 在目录中增删文件
- **x（执行）**：运行程序 / 进入目录

可通过 `ls -l` 查看权限，例如：
```bash
-rw-r--r-- 1 yubao dev 1024 Apr 4 10:00 file.txt
```
表示：`yubao`（所有者）可读写，`dev` 组成员只读，其他人只读。

关于文件/目录权限的修改，在后续文件管理学习详细介绍。

### 六、举例

用户/用户组不仅可以配置对于文件/目录的权限，还可以配置执行命令的权限。

当我登录`corey`账号，想执行`sudo fdisk -l` 命令，查看硬盘分区情况时，会出现如下情况：

以上提示表明，`corey`用户无法执行`sudo`指令，这就需要进行权限配置。并且，这次试图越权限的操作将会被报告，保存在日志文件中：`/var/log/secure`：

![04-securelog](E:\Note\HuaweiLearningCourse\LinuxLearningCourse\image\01-用户管理\02-securelog.png)

可以通过配置`\etc\sudoers`文件配置执行`sudo`命令的权限。（可配置的文件非常多，通过实践过程进行积累）

要给用户 `corey` 添加 `sudo` 权限，你需要以 **root** 身份操作。核心步骤是修改 `/etc/sudoers` 文件。

> ⚠️ **重要提示**：`/etc/sudoers` 是一个非常关键的系统文件，语法错误可能导致所有用户都无法使用 `sudo`。因此，**强烈建议使用 `visudo` 命令来编辑它**，因为 `visudo` 会在保存时检查语法错误，防止你保存一个损坏的配置文件。

以下是详细的操作步骤：

**步骤 1: 切换到 root 用户**

```bash
[corey@centOS76 ~]$ su - root
# 输入 root 密码
```

**步骤 2: 使用 `visudo` 命令编辑 sudoers 文件**

在 root 提示符下，输入：
```bash
[root@centOS76 ~]$ visudo
```
这会用默认的文本编辑器（通常是 `vi`）打开 `/etc/sudoers` 文件。

**步骤 3: 在文件中添加授权规则**

1.  在 `visudo` 打开的文件中，找到如下这一行：
    ```conf
    ## Allow root to run any commands anywhere
    root    ALL=(ALL)       ALL
    ```

2.  **在这行下面**，添加一行新的规则，将 `corey` 替换为你的用户名：
    ```conf
    corey   ALL=(ALL)       ALL
    ```
    这行规则的含义是：
    - `corey`: 用户名。
    - `ALL=(ALL)`: 允许该用户从任何主机（第一个 `ALL`）以任何其他用户的身份（括号内的 `ALL`）执行命令。
    - `ALL`: 允许执行任何命令。

    > **更简单的方法（推荐）**：许多现代 Linux 发行版（包括 CentOS 7）都预定义了一个名为 `wheel` 的用户组，并且已经有一行被注释掉的规则来授权这个组的所有成员。你可以直接启用它。
    >
    > 在文件中找到这一行（通常在靠近顶部的位置）：
    > ```conf
    > ## Allows people in group wheel to run all commands
    > # %wheel        ALL=(ALL)       ALL
    > ```
    > 删除行首的 `#` 号来取消注释，使其生效：
    > ```conf
    > %wheel        ALL=(ALL)       ALL
    > ```
    > 然后，你只需要将 `corey` 用户添加到 `wheel` 组即可（见步骤4）。这种方法更易于管理多个管理员用户。

**步骤 4: （如果采用 `wheel` 组方法）将用户 `corey` 添加到 `wheel` 组**

如果你选择了上面提到的 `wheel` 组方法，在保存并退出 `visudo` 后，还需要执行以下命令：
```bash
[root@centOS76 ~]# usermod -aG wheel corey
```
- `-aG` 选项表示将用户追加（`-a`）到指定的附加组（`-G`）中，而不会影响其现有的组。

**步骤 5: 保存并退出**

- 如果你使用的是 `vi` 编辑器：
    1.  按 `Esc` 键确保你处于命令模式。
    2.  输入 `:wq` 并按回车。`visudo` 会自动检查语法，如果没问题就会保存并退出。

**步骤 6: 验证权限**

1.  退出 root shell，切换回 `corey` 用户：
    ```bash
    [root@centOS76 ~]# exit
    [corey@centOS76 ~]$
    ```

2.  尝试使用 `sudo` 执行一个命令，例如：
    ```bash
    [corey@centOS76 ~]$ sudo whoami
    ```
    系统会提示你输入 `corey` 自己的密码。输入正确后，如果返回 `root`，则说明权限已成功添加！

现在，用户 `corey` 就拥有了完整的 `sudo` 权限。
