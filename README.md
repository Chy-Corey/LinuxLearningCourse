本人此前已学习过 Linux，并在两次实习期间积累了实际使用经验，熟悉常用命令、文件系统操作及基础服务配置。此次入职后，根据新员工培训要求，需完成软件开发方向的学习计划，其中包含一门系统的 Linux 课程。尽管已有一定基础，但借此机会重温核心概念、查漏补缺，将有助于夯实技术根基，更好地融入团队开发流程。

在初次学习 Linux 时，由于此前仅使用过 Windows 系统，我对 Linux 中许多看似“繁琐”的操作感到困惑——例如为什么要手动创建用户、分配权限，为什么要通过命令行配置 IP 地址、路由表和网关，甚至为什么连安装软件都需要理解依赖关系和包管理机制。在 Windows 图形界面“开箱即用”的使用习惯下，这些操作显得格外复杂且不直观。随着后续的学习和实践，我才逐渐意识到，这种“手动”并非多余，而是 Linux 设计哲学的体现——**透明、可控、模块化**。它赋予用户对系统的完全掌控权，也要求使用者理解底层原理。正是这种设计，使得 Linux 在服务器、云计算和嵌入式等场景中具备极高的稳定性、安全性和灵活性。如今回看，当初那些“复杂的管理操作”，恰恰是深入掌握操作系统本质的关键入口。

在撰写学习记录时，我会适当将 Linux 与 Windows 进行对比：先通过两者在设计理念和使用方式上的差异，理解“为什么 Linux 需要这些功能”；在此基础上，再深入学习具体的实现方法和操作命令。这样的对比式学习，有助于从“知其然”走向“知其所以然”，从而更扎实地掌握 Linux 的核心机制。
