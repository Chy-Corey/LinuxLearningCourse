## 网络管理

Linux 网络 = TCP/IP 协议栈在操作系统中的实现

Linux 的网络子系统是 **TCP/IP 协议栈的内核级实现**，遵循 **OSI 模型**（简化为 5 层）或更实用的 **TCP/IP 四层模型**：

| TCP/IP 层      | 对应 Linux 内核模块/概念                | 典型命令/工具                |
| -------------- | --------------------------------------- | ---------------------------- |
| **应用层**     | 用户进程（如 `nginx`, `sshd`）          | `curl`, `telnet`, `lsof`     |
| **传输层**     | `sock` 结构体、端口绑定、TCP/UDP 状态机 | `ss`, `netstat`              |
| **网络层**     | IP 路由表、ARP 表、ICMP、IP 分片        | `ip route`, `arp -a`, `ping` |
| **数据链路层** | 网卡驱动、MAC 地址、以太网帧            | `ip link`, `ethtool`         |
| **物理层**     | 网线、光模块、PHY 芯片（硬件抽象）      | —                            |

> ✅ 所有 Linux 网络命令，本质都是对这五层中某一层状态的读取或修改。

### 1. 网络基础

#### 1.1 Linux 网络术语 → 对应计算机网络概念

| Linux 术语                | 计算机网络原理对应      | 说明                                                         |
| ------------------------- | ----------------------- | ------------------------------------------------------------ |
| **网络接口（interface）** | 数据链路层实体（NIC）   | 如 `eth0` 是一个以太网适配器，对应 OSI L2 设备；每个接口有 MAC 地址（48 位，全球唯一） |
| **IP 地址**               | 网络层地址（IPv4/IPv6） | 用于跨网络寻址；**子网划分**基于 CIDR（如 `/24`），本质是“网络前缀 + 主机后缀” |
| **端口（port）**          | 传输层多路复用标识符    | UDP/TCP 使用 16 位端口号区分同一主机上的不同服务（0~65535）；**知名端口**（0–1023）需 root 权限绑定 |
| **套接字（socket）**      | 应用层与传输层的接口    | 五元组：`(源IP, 源端口, 目的IP, 目的端口, 协议)` 唯一标识一个连接（如 `192.168.1.10:54321 ↔ 8.8.8.8:53`） |
| **回环接口 `lo`**         | 本地环回（Loopback）    | 不经过物理网卡，直接在内核中转发；用于本机服务测试（如 `localhost:8080`） |

> 🔍 **关键点**：  
> - Linux 中“一切皆文件”，网络接口也映射为 `/sys/class/net/eth0/` 下的属性文件。  
> - `ifconfig` 已废弃，因它无法处理现代复杂网络（如 VLAN、隧道、命名空间），而 `ip` 命令直接操作内核 netlink 接口，更底层、更强大。

#### 1.2 IP 的分类 → 从 IPv4 地址结构到现代实践

###### （1）传统 A/B/C 类地址（已淘汰，但需理解历史）
| 类别 | 首字节范围 | 网络位/主机位   | 示例               |
| ---- | ---------- | --------------- | ------------------ |
| A 类 | 1–126      | /8（24位主机）  | `10.0.0.1/8`       |
| B 类 | 128–191    | /16（16位主机） | `172.16.0.1/16`    |
| C 类 | 192–223    | /24（8位主机）  | `192.168.1.100/24` |

> ⚠️ 问题：地址浪费严重（如 C 类最多 254 台主机，但很多局域网只需 10 台）→ 引出 **CIDR（无类别域间路由）**

###### （2）CIDR 与子网掩码（现代标准）
- `/24` = `255.255.255.0` → 网络地址：`192.168.1.0`，广播地址：`192.168.1.255`
- **可变长子网掩码（VLSM）**：允许在一个大网段内切分子网（如 `/24` 切成两个 `/25`）

###### （3）特殊地址类型（RFC 1918 私有地址）
| 范围             | 用途                      |
| ---------------- | ------------------------- |
| `10.0.0.0/8`     | 大型私有网络              |
| `172.16.0.0/12`  | 中型私有网络              |
| `192.168.0.0/16` | 小型局域网（家庭/办公室） |

> 💡 **为什么需要私有地址？**  
> IPv4 地址枯竭 → NAT（网络地址转换）技术诞生：内网私有地址通过路由器映射为公网地址通信（**NAT 在 Linux 中由 `iptables`/`nftables` 实现**）。

###### （4）IPv6 简介（补充）
- 128 位地址（`2001:db8::1`），支持自动配置（SLAAC）、无 NAT（端到端通信）
- Linux 中通过 `ip -6 addr` 查看 IPv6 地址

### 2. 配置网络接口

#### 2.1 使用命令配置网络接口 → 对应数据链路层与网络层交互

```bash
# 添加 IP 地址（网络层配置）
sudo ip addr add 192.168.1.100/24 dev eth0
```

| 部分               | 含义                                                         |
| ------------------ | ------------------------------------------------------------ |
| `sudo`             | 以管理员权限运行（因为修改网络配置需要 root 权限）           |
| `ip addr add`      | 调用 `ip` 工具的子命令，用于向网络接口添加 IP 地址           |
| `192.168.1.100/24` | 要添加的 IP 地址 + 子网前缀长度（CIDR 表示法）               |
| `dev eth0`         | 指定操作的目标网络接口为 `eth0`，`dev` 是 “device”（设备）的缩写，用于指定操作的目标网络接口（网卡）。 |

**背后发生了什么？**

1. 内核创建一个 `struct in_ifaddr`（IPv4 地址条目）
2. 向 ARP 表添加该 IP 的“本地绑定”记录（避免发 ARP 请求自己）
3. 若启用 `arp_filter`，还会检查是否属于本机子网
4. 触发 `NETDEV_UP` 事件，通知用户空间（如 NetworkManager）
5. 自动配置直链路由，执行此命令后，内核会**自动添加一条直连路由（directly connected route）** 到路由表中：

   ```bash
   $ ip route show
   192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
   ```

   这条路由的含义是：

   > “所有发往 `192.168.1.0/24` 网段的数据包，都直接通过 `eth0` 接口发送（不需要经过网关）。”

   这使得本机可以：

   - 直接与 `192.168.1.1` ~ `192.168.1.254` 的设备通信（通过 ARP 获取 MAC 地址）
   - 发送广播到 `192.168.1.255`

> 📌 **关键协议联动**：
> - **ARP（地址解析协议）**：当发送数据包时，若目的 IP 不在同一子网，需查路由表 → 找到网关 → 发 ARP 请求获取网关 MAC。
> - `ip addr show` 中的 `scope global` 表示该地址可用于全局通信；`scope host` 仅限本机（如 `127.0.0.1`）。

**示例：配置默认网关**

```bash
sudo ip route add default via 192.168.1.1 dev eth0
```
- 这是在**路由表（FIB，Forwarding Information Base）**中添加一条默认路由
- 内核使用最长前缀匹配（LPM）查找路由：先匹配 `/32` → `/24` → 最终匹配 `0.0.0.0/0`

> 🔁 **路由决策流程**（数据包出站时）：
> ```
> 应用层 → socket → IP 层 → 查路由表 → 选出口设备 → 查 ARP 表 → 构造以太网帧 → 驱动发往 PHY
> ```

#### 2.2 手动编辑文件配置网络接口 → 持久化配置与启动流程

以 Ubuntu 的 Netplan 为例：
```yaml
# /etc/netplan/01-network-manager.yaml
network:
  version: 2
  renderer: networkd   # 或 NetworkManager
  ethernets:
    eth0:
      dhcp4: no
      addresses: [192.168.1.100/24]
```

###### ▶ 启动时发生了什么？
1. 系统引导 → `systemd` 启动 `systemd-networkd` 或 `NetworkManager`
2. 读取配置文件 → 调用 `libnl`（Netlink 库）→ 向内核发送 `RTM_NEWADDR`, `RTM_NEWROUTE` 消息
3. 内核更新 `rtable`（路由表）和 `neigh_table`（邻居表）
4. 触发 udev 事件，可能激活 DHCP 客户端（如 `dhclient`）

> 🧠 **计算机网络视角**：  
> 这相当于在**网络层和数据链路层**之间建立了一套“静态配置数据库”，替代了动态协议（如 DHCP），适用于服务器等需固定 IP 的场景

### 3. 配置路由

#### 3.1 查看路由表 → 理解 IP 转发机制

```bash
ip route show
# 输出示例：
default via 192.168.1.1 dev eth0 proto static 
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
```

###### ▶ 路由表条目字段含义：
| 字段              | 含义                  | 网络原理                                             |
| ----------------- | --------------------- | ---------------------------------------------------- |
| `default`         | 默认路由（0.0.0.0/0） | 当无更精确匹配时使用，类似“兜底规则”                 |
| `via 192.168.1.1` | 下一跳地址            | **必须可达**（通过 ARP 解析其 MAC）                  |
| `dev eth0`        | 出接口                | 决定从哪个网卡发出                                   |
| `proto kernel`    | 来源协议              | 内核自动生成的直连路由（直连子网）                   |
| `scope link`      | 作用域                | `link` 表示仅限本链路（无需路由）；`global` 可跨网段 |

> 📐 **路由查找算法**（Linux 内核）：
> - 使用 **FIB（Forwarding Information Base）**，基于哈希+树结构加速查找
> - 支持策略路由（`ip rule`），可按源地址、TOS 等选择不同路由表

#### 3.2 新增/删除路由 → 控制数据流向

```bash
# 添加静态路由（访问 10.0.0.0/8 经过 192.168.1.254）
sudo ip route add 10.0.0.0/8 via 192.168.1.254 dev eth0

# 删除默认路由
sudo ip route del default
```

###### ▶ 应用场景与网络原理：
- **多网卡出口分流**：如 eth0 上网，eth1 连内网 → 为内网流量加特定路由
- **虚拟专用网络（VPN）**：OpenVPN 创建 `tun0` 接口，并添加路由指向隧道
- **容器网络**：Docker 为每个容器创建 veth pair，并添加路由使容器可通信

> ⚙️ **注意**：添加路由 ≠ 自动配置 ARP！若下一跳不可达，仍会失败。需确保：
> - 下一跳 IP 存在且在线
> - 本地接口已 UP
> - 防火墙未丢弃 ICMP/ARP

### 4. 网络侦测

#### 4.1 网络侦测命令 → 模拟协议交互过程

| 命令         | 模拟的协议行为                 | 说明                                                        |
| ------------ | ------------------------------ | ----------------------------------------------------------- |
| `ping`       | ICMP Echo Request/Reply        | 测试**网络层可达性**（不涉及传输层端口）                    |
| `traceroute` | 发送 TTL 递增的 UDP/ICMP 包    | 利用 **TTL 超时产生 ICMP Time Exceeded** 报文，逐跳探测路径 |
| `mtr`        | `ping` + `traceroute` 实时合并 | 动态显示每跳丢包率与延迟，适合诊断不稳定链路                |
| `ss -tuln`   | 读取内核 socket 表             | 显示所有监听/连接的套接字（五元组），反映**传输层状态**     |

> 🌐 **深入 `ping` 工作流程**：
> 1. 应用层调用 `socket(AF_INET, SOCK_RAW, IPPROTO_ICMP)`
> 2. 构造 ICMP Echo Request 报文（Type=8, Code=0）
> 3. 内核填充 IP 头（源/目的 IP）、计算校验和
> 4. 查路由表 → 得到出口设备 → 查 ARP 表 → 获取目的 MAC
> 5. 封装以太网帧（DA=目标 MAC, SA=本机 MAC, Type=0x0800）
> 6. 驱动发往网卡 → 物理层发送

#### 4.2 侦测目的节点是否可达 → 三层 vs 四层可达性

| 方法                  | 检测层级          | 能否穿透防火墙？   | 说明                              |
| --------------------- | ----------------- | ------------------ | --------------------------------- |
| `ping`                | 网络层（ICMP）    | ❌ 通常被防火墙丢弃 | 仅说明 IP 层可达                  |
| `nc -zv host port`    | 传输层（TCP SYN） | ✅ 可检测端口开放   | 发送 SYN，收到 SYN-ACK 则端口开放 |
| `telnet host port`    | 传输层            | ✅                  | 人工交互式测试                    |
| `curl -I http://host` | 应用层            | ✅                  | 测试 HTTP 服务是否响应            |

> 🔒 **安全提示**：  
> - `ping` 不通 ≠ 服务不可用（可能是 ICMP 被禁）  
> - 真正的“可达性”应定义为：**目标端口能建立 TCP 连接并完成应用层握手**

#### 4.3 侦测到达目的节点的路径 → 路由追踪原理

```bash
traceroute 8.8.8.8
```

###### ▶ 工作原理（UDP 模式）：
1. 发送 UDP 包，TTL=1 → 第一跳路由器返回 **ICMP Time Exceeded**
2. TTL=2 → 第二跳返回 ICMP Time Exceeded
3. ……
4. 当包到达目的主机，因目的端口未开放，返回 **ICMP Port Unreachable**
5. traceroute 收到此报文，判定路径结束

> 📉 **为什么有时显示 `* * *`？**  
> 中间路由器配置了 **不响应 ICMP 错误报文**（安全策略），导致超时。

###### ▶ 替代方案：
- `mtr`：持续发送探测包
- `traceroute -T -p 80`：用 TCP SYN 替代 UDP，绕过 UDP 过滤
- `hping3`：自定义 TCP/ICMP 包进行高级探测

### 举例

场景设定：你家和朋友家不在同一个小区

想象一下：

- **你（小明）** 住在 **阳光小区 A 栋 101 室**
- **你的朋友（小红）** 住在 **月亮小区 C 栋 202 室**
- 两个小区之间**没有直连道路**，必须通过 **城市主干道** 才能到达

> 🔑 这个比喻中：
> - **每个小区 = 一个子网（如 `192.168.1.0/24`）**
> - **你家地址 = IP 地址（如 `192.168.1.100`）**
> - **小区门口的保安亭/出口 = 网关（Gateway）**
> - **城市交通系统 = 路由器（Router）**

问题来了：你怎么去小红家？

#### 情况 1️⃣：你们在同一个小区（同一子网）

- 比如小红也住在阳光小区 B 栋 303。
- 你直接走过去就行，**不需要经过小区大门**。
- ✅ **对应网络行为**： 
  你的电脑发现目标 IP（如 `192.168.1.50`）和自己在同一子网（`192.168.1.0/24`），于是直接通过 **ARP 协议** 获取对方 MAC 地址，**二层直接通信**。

> 💡 此时**不需要路由**！

#### 情况 2️⃣：小红在另一个小区（不同子网）✅ ← 这就是需要路由的场景！

你想去月亮小区，但：
- 你不知道怎么走
- 你不能翻墙过去
- 你只能先走出自己小区

于是你问保安：“**怎么去月亮小区？**”

保安说：“出门右转，上**城市主干道**，看到‘月亮小区’路牌就到了。”

> 🔑 **这个“保安”就是你的默认网关（Default Gateway）！**

#### 对应到真实网络

假设你的电脑配置如下：
```bash
IP 地址:     192.168.1.100
子网掩码:    255.255.255.0   → 子网是 192.168.1.0/24
默认网关:    192.168.1.1     ← 这是路由器的内网接口 IP
```

现在你要访问百度服务器（`180.101.49.12`）或朋友的电脑（`192.168.2.50`）。

**第一步：判断目标是否在同一子网？**

- 目标 `192.168.2.50` & `255.255.255.0` = `192.168.2.0`
- 你的网络地址 = `192.168.1.0`
- ❌ 不同！所以**不能直接通信**

**第二步：怎么办？→ 查看“路由表”（你的出行计划本）**

运行命令查看路由表：
```bash
ip route show
```

输出可能包含：
```
default via 192.168.1.1 dev eth0    ← 默认路由（最关键！）
192.168.1.0/24 dev eth0 proto kernel scope link src 192.168.1.100
```

解读：

- **`default via 192.168.1.1`** 的意思是： 
  “所有**不属于本地子网**的目标地址，请把数据包发给 `192.168.1.1`（网关）处理！”

**第三步：你的电脑把数据包发给网关（192.168.1.1）**

- 你的电脑通过 ARP 获取 `192.168.1.1` 的 MAC 地址
- 把目标为 `192.168.2.50` 的 IP 包封装成帧，发给网关

**第四步：路由器（网关）收到后，查自己的路由表**

假设路由器知道：
```
192.168.2.0/24 via 10.0.0.2 dev eth1
```
意思是：“要去 `192.168.2.0/24` 网段，请从 `eth1` 接口发往下一跳 `10.0.0.2`”

于是路由器把包转发出去……最终到达小红的电脑。

**路由配置的核心作用总结**

| 作用                            | 说明                                                     |
| ------------------------------- | -------------------------------------------------------- |
| **1. 决定数据包的“下一步去哪”** | 就像导航告诉你“前方500米右转”                            |
| **2. 实现跨网络通信**           | 没有路由，你只能和同子网设备通信（就像只能在小区里活动） |
| **3. 避免广播风暴**             | 路由器隔离广播域，不会把 ARP 请求发到其他网络            |
| **4. 支持复杂网络拓扑**         | 多条路径、负载均衡、故障切换都靠路由协议实现             |

**如果没有路由配置，会发生什么？**

- 你的电脑尝试直接发包给 `192.168.2.50`
- 但它在本地网络找不到这个 IP（因为根本不在同一个物理网络）
- 结果：**“Destination Host Unreachable”**（目标主机不可达）
- 你 ping 不通，ssh 连不上，网页打不开……

> 💡 就像你站在小区里大喊“小红！”，但她在另一个小区——她根本听不见！

**补充：除了默认路由，还有其他路由吗？**

是的！比如：

```bash
# 添加一条静态路由：去 10.10.0.0/16 网段，走 192.168.1.254
sudo ip route add 10.10.0.0/16 via 192.168.1.254
```

这就像你告诉家人：
> “如果要去‘科技园区’，不要走主干道，走东门那条小路更快！”

## 🧠 最后一句话总结

> **路由 = 网络世界的地图和导航。没有它，你的数据包就像迷路的人，永远到不了目的地。**

而“路由配置”就是**给每台设备（尤其是路由器）装上正确的地图**，让它知道：  
“遇到去某地的包裹，该交给谁送？”
