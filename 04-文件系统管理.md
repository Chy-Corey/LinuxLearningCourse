文件系统管理

**文件系统**就是操作系统用来管理存储设备（如硬盘、SSD、U盘）上数据的**方法和数据结构**。

你可以把它想象成一位**“图书管理员”**。如果没有它，硬盘就是一堆杂乱无章的字节，数据存进去就找不到了；有了它，数据才能被有序地组织、命名、存储和检索。

### 核心定义与功能

文件系统不仅仅是把数据存进去，它主要解决了以下四个核心问题：

1.  **数据组织（怎么放）：** 它将数据分割成块，并用**目录（文件夹）**和**树状结构**来组织文件，让你能直观地分类管理。
2.  **命名空间（叫什么）：** 它规定了文件的命名规则（比如文件名长度、允许字符），并通过**路径**（如 `C:\Users\Doc` 或 `/home/user`）来唯一定位文件。
3.  **元数据管理（属性）：** 除了文件内容，它还要记录文件的“档案”，比如**大小、创建时间、修改时间、权限**（谁能读、谁能写）以及文件在磁盘上的物理位置。
4.  **存储空间管理（剩多少）：** 它负责分配和回收磁盘空间。当你删除文件时，文件系统会将该空间标记为“空闲”，以便下次使用。

### 工作原理

在 Linux/Unix 等系统中，文件系统的工作依赖于几个关键的数据结构（Windows 的 NTFS 也有类似的机制，如 MFT）：

*   **超级块（Superblock）：** 相当于文件系统的“总控制台”，记录了整个分区的大小、空闲块数量等全局信息。
*   **索引节点（Inode）：** 这是文件系统的核心。每个文件都有一个唯一的 Inode，里面存储了文件的**元数据**（权限、所有者、大小、数据块指针），但**不包含文件名**。
*   **目录项（Directory Entry）：** 目录本质上是一个文件，它的作用是建立**文件名**与 **Inode** 之间的映射关系。
*   **数据块（Data Block）：** 实际存储文件内容的地方。
*   图解：<img src=".\image\04-文件系统管理\01-inode超级块.png" alt="01-inode超级块" style="zoom:50%;" />

**当你打开一个文件时，流程大致如下：**
1.  系统解析路径，在目录中找到文件名。
2.  通过文件名找到对应的 Inode 号。
3.  读取 Inode，获取文件数据在磁盘上的具体位置指针。
4.  去数据块读取实际内容。

### 常见的文件系统类型

不同的操作系统和应用场景需要不同的文件系统。以下是主流类型的对比：

| 文件系统  | 适用平台   | 特点与优势                                                   | 局限性                                                      |
| :-------- | :--------- | :----------------------------------------------------------- | :---------------------------------------------------------- |
| **NTFS**  | Windows    | 现代 Windows 默认格式。支持超大文件、权限控制、加密、压缩和**日志功能**（断电不易丢数据）。 | 在 macOS/Linux 上通常只能读，写入需要第三方软件。           |
| **FAT32** | 通用 (U盘) | 兼容性极好，几乎所有设备（电视、游戏机、相机）都能读。       | **不支持超过 4GB 的单个文件**，分区最大仅 2TB，无权限管理。 |
| **exFAT** | 闪存设备   | 专为 U 盘和 SD 卡设计。兼容性好（Win/Mac 通吃），且突破了 4GB 文件限制。 | 相比 NTFS 缺少日志功能，数据恢复较难，不适合做系统盘。      |
| **ext4**  | Linux      | Linux 主流默认格式。稳定、支持日志、处理大量小文件效率高。   | Windows 默认无法识别。                                      |
| **APFS**  | macOS      | 苹果设备专用。针对 SSD 优化，支持快照、克隆，速度极快。      | 仅限苹果生态使用。                                          |

### 为什么文件系统很重要？

1.  **数据完整性：** 现代文件系统（如 NTFS, ext4）都有**日志（Journaling）**功能。它能防止断电导致的数据错乱，确保文件系统结构的一致性。
2.  **安全性：** 通过权限管理（如 Linux 的 rwx 权限或 Windows 的 ACL），防止用户误删系统文件或窃取他人数据。
3.  **抽象化：** 它让你不需要知道数据具体存在硬盘的哪个磁道或扇区，只需要通过简单的文件名和文件夹就能操作海量数据。

总结来说，文件系统是连接**用户/应用程序**与**物理存储硬件**之间的桥梁，没有它，数字世界将是一盘散沙。

### 举例对比：ext2 vs ext4

我们可以把文件系统比作一个**“图书管理员”**，把硬盘比作**“书架”**。

在 Linux 世界中，ext2 是老一代管理员，ext4 是新一代管理员。它们最大的区别就在于有没有**“记账本”（日志机制，Journaling）**。

#### 1. ext2：老实但“记性差”的管理员（无日志）
ext2 是最早期的 Linux 文件系统，它的设计很简单：
*   **工作方式**：当你让它保存一个文件时，它会直接去书架上修改。它会先写数据（书的内容），然后更新目录卡片（元数据，比如书名、位置）。
*   **断电后果**：
    如果在它写到一半时突然断电（比如书刚放上去，目录卡片还没来得及写），ext2 就会“断片儿”。
    *   **现象**：来电后，ext2 根本不知道刚才发生了什么。为了安全起见，它必须把整个图书馆（硬盘）的所有书和卡片全部检查一遍（这就是著名的 `fsck` 过程）。
    *   **代价**：如果硬盘很大（比如几 TB），这个检查过程可能长达数小时，期间你无法使用系统。而且，如果运气不好，刚才写了一半的数据可能就彻底丢失或变成乱码了。

#### 2. ext4：精明且“有记性”的管理员（有日志）
ext4 是现代 Linux 的主流文件系统，它引入了**日志机制（Journaling）**。
*   **工作方式**：它在动手之前，会先在一个专门的**“记账本”（日志区）**上写下计划。
    *   **步骤**：
        1.  **写日志**：在记账本上写下：“我要在 A 位置写入数据 X，并更新目录卡片 Y”。
        2.  **写数据**：去书架上真正写入数据。
        3.  **提交**：在记账本上打个勾，表示“完成了”。
*   **断电后果**：
    如果在步骤 2 或 3 之间断电了：
    *   **现象**：来电后，ext4 不需要检查整个图书馆。它只需要翻开“记账本”**。
    *   **恢复**：它会看记账本，发现有一条记录没打勾。于是它会根据记录，把没做完的事情快速做完（重放日志），或者把做了一半的事情撤销（回滚）。
    *   **优势**：这个过程通常只需要**几秒钟**，而且能保证文件系统结构的一致性，不会出现“有目录没文件”的尴尬情况。

#### 直观对比表

| 特性           | ext2 (无日志)                                      | ext4 (有日志)                                    |
| :------------- | :------------------------------------------------- | :----------------------------------------------- |
| **断电应对**   | **灾难级**。必须全盘扫描检查错误。                 | **秒级恢复**。只检查日志，快速回滚或重做。       |
| **数据一致性** | **差**。容易出现“孤儿文件”（有数据没名字）或坏块。 | **强**。保证元数据的一致性，文件系统结构不损坏。 |
| **适用场景**   | 现在的 U 盘、嵌入式设备（因为写入少，速度快）。    | 现代服务器、电脑硬盘（数据量大，要求安全）。     |
| **形象比喻**   | 直接在墙上写字，停电了就写废了。                   | 先在草稿纸上写好，确认无误再抄到墙上。           |

#### 文件系统的作用：通过这个例子我们学到了什么？

通过 ext2 和 ext4 的对比，我们可以看到文件系统在“断电”这个场景下发挥了至关重要的作用：

1.  **秩序维护者（元数据管理）**
    文件系统不仅仅是存数据，更重要的是维护**元数据**（目录结构、文件大小、权限）。ext4 的日志功能主要保护的就是元数据。它确保了即使数据丢了，文件系统的“目录索引”依然是清晰的，不会导致整个系统崩溃。

2.  **容错与恢复（可靠性）**
    文件系统通过**日志（Journaling）**这种“写前记录”的策略，将“随机发生的断电风险”转化为了“可控的事务处理”。它保证了操作要么**完全成功**，要么**完全回滚**，不会停留在中间那种“半死不活”的状态。

3.  **性能与安全的平衡**
    ext2 虽然快（因为少了一步写日志的操作），但风险太大；ext4 虽然多了一步写日志，稍微牺牲了一点点写入速度，但换来了巨大的数据安全保障。现代文件系统（如 NTFS, APFS, ext4）都选择了**带日志**的路线，这就是文件系统进化的方向。

**总结：**文件系统就像是你数据的**保险箱**。ext2 是那种老式挂锁，一旦钥匙丢了（断电）或者卡住了，你可能得把整个箱子撬开检查；而 ext4 是带有备用钥匙和自动复位功能的智能锁，遇到意外它能迅速恢复到安全状态，确保你的数据井井有条。

## 磁盘分区介绍

Linux 磁盘分区是文件系统管理中非常基础且重要的一环。它不仅仅是把硬盘“切开”那么简单，更涉及到数据管理、系统性能以及启动机制。

### 1. 为什么要进行磁盘分区？

磁盘分区就是将一块物理硬盘在逻辑上划分为多个独立的存储单元。这样做主要有以下几个目的：
*   **数据隔离与安全**：如果系统崩溃需要重装，可以只格式化根分区 `/`，而保留 `/home` 分区中的用户数据不丢失。
*   **性能优化**：将频繁读写的日志文件（`/var`）或临时文件（`/tmp`）单独分区，可以减少磁盘碎片对系统文件的影响。

### 2. 核心概念：三种分区类型

在传统的 MBR 分区模式下，分区主要分为三类，理解它们的关系至关重要：

| 分区类型                | 说明                                   | 特点                                                         |
| :---------------------- | :------------------------------------- | :----------------------------------------------------------- |
| **主分区 (Primary)**    | 直接建立在硬盘上的分区。               | **数量有限**：一块硬盘最多只能有 **4个** 主分区。系统引导通常必须放在主分区中。 |
| **扩展分区 (Extended)** | 为了解决主分区数量限制而存在的“容器”。 | **不能存储数据**：它本身不能被格式化或挂载，仅用于容纳逻辑分区。一块硬盘最多只能有 **1个** 扩展分区。 |
| **逻辑分区 (Logical)**  | 建立在扩展分区内部的分区。             | **数量灵活**：理论上数量不限（取决于操作系统）。我们通常将数据存储在逻辑分区中。 |

**💡 经典组合策略**：
如果你需要 6 个分区，通常的做法是：**3个主分区 + 1个扩展分区**（在这个扩展分区里再划分出 3 个逻辑分区）。

### 3. 设备命名规则

在 Linux 中，一切皆文件，硬盘也被视为文件，存放在 `/dev` 目录下。

*   **硬盘标识**：
    *   `/dev/sda`：系统中检测到的**第一块** SCSI 或 SATA 硬盘。
    *   `/dev/sdb`：**第二块**硬盘，以此类推。
    *   （注：早期的 IDE 硬盘通常标识为 `/dev/hda`，现在较少见）
*   **分区标识**：
    *   `/dev/sda1`：第一块硬盘的**第1个分区**。
    *   `/dev/sda5`：第一块硬盘的**第1个逻辑分区**（逻辑分区编号总是从 5 开始）。

### 4. 分区表标准：MBR 与 GPT

这是现代 Linux 管理中必须注意的区别，决定了你能用多大的硬盘。

*   **MBR (主引导记录)**：
    *   **老旧标准**：支持最大磁盘容量仅为 **2TB**。
    *   **限制**：最多支持 4 个主分区。
    *   **工具**：通常使用 `fdisk` 进行分区。
*   **GPT (全局唯一标识符分区表)**：
    *   **现代标准**：支持 **2TB 以上** 的超大磁盘（理论上限极高）。
    *   **优势**：支持更多的分区数量（Linux下通常支持128个主分区，无需扩展分区概念），且数据冗余性更好（头部和尾部都有备份）。
    *   **工具**：通常使用 `parted` 或 `gdisk` 进行分区。

### 5. 常用分区与管理工具

在 Linux 命令行下，你经常会用到以下工具：

*   **`fdisk`**：
    *   最经典的命令行分区工具。
    *   **适用**：MBR 分区表（不支持 2TB 以上磁盘的 GPT 分区）。
    *   **常用命令**：`fdisk -l`（查看磁盘信息），`fdisk /dev/sdb`（对 `sdb` 分区）。
*   **`parted`**：
    *   功能强大的高级分区工具。
    *   **适用**：支持 MBR 和 **GPT**，适合处理大硬盘。
*   **`lsblk`**：
    *   以树状图列出所有块设备（磁盘及分区）的信息，非常直观，常用于查看挂载点。
*   **`mkfs`**：
    *   分区完成后，用于**格式化**分区（创建文件系统），例如 `mkfs.ext4 /dev/sdb1`。
*   **`mount`**：
    *   用于将分区**挂载**到目录树上，使其可以被访问。

### 6. 常见的分区规划建议

在安装 Linux 系统时，一个合理的分区方案通常包含：

1.  **`/boot`** (200MB - 1GB)：存放启动文件（内核、引导程序）。
2.  **`/` (根分区)** (10GB - 50GB+)：存放系统软件和配置文件。
3.  **`swap` (交换分区)**：虚拟内存，通常建议为物理内存的 1-2 倍（视内存大小而定），用于内存不足时的数据交换。
4.  **`/home`** (剩余空间)：存放用户个人数据，建议单独分区以便系统重装时保留数据。

## 磁盘分区命令

### 1. 创建磁盘分区

在 Linux 中创建磁盘分区，主要使用两个命令行工具：**`fdisk`**（适用于小于 2TB 的磁盘）和 **`parted`**（支持大容量磁盘，推荐用于 ≥2TB 或使用 GPT 分区表的场景）。

以下是详细操作步骤（以 root 身份执行）：

**第一步：确认目标磁盘**

先列出所有磁盘设备，找到你要分区的新磁盘（**务必确认设备名，避免误操作系统盘！**）：

```bash
lsblk
# 或
fdisk -l
```

假设你的新磁盘是 `/dev/sdb`（没有分区、未挂载）。

方法一：使用 `fdisk`（适合 ≤2TB 磁盘，MBR 分区表）

```bash
fdisk /dev/sdb
```

进入交互界面后，按以下步骤操作：

1. 输入 `n` → 创建新分区  
2. 选择分区类型：
   - `p` → 主分区（最多4个）
   - `e` → 扩展分区（用于创建逻辑分区）
3. 分区号：默认回车（通常为1）
4. 起始扇区：默认回车
5. 结束扇区：输入大小（如 `+10G` 表示10GB，或直接回车用满剩余空间）
6. （可选）输入 `t` 修改分区类型（如 Linux LVM、swap 等）
7. 输入 `w` → **写入更改并退出**（⚠️ 此步会真正修改磁盘！）

> ✅ 示例：创建一个主分区占满整个磁盘
> ```
> Command (m for help): n
> Partition type: p
> Partition number: 1
> First sector: (默认)
> Last sector: (默认)
> Command (m for help): w
> ```

方法二：使用 `parted`（支持 GPT 和大磁盘）

```bash
parted /dev/sdb
```

交互操作如下：

1. 创建分区表（**会清空磁盘！**）：
   ```bash
   (parted) mklabel gpt    # 推荐 GPT（支持 >2TB）
   # 或
   (parted) mklabel msdos  # MBR（兼容旧系统，≤2TB）
   ```
2. 创建分区：
   ```bash
   (parted) mkpart primary ext4 0% 100%
   # 说明：分区名为 primary，文件系统类型 ext4（仅提示，不格式化），从0%到100%
   ```
3. 查看分区：
   ```bash
   (parted) print
   ```
4. 退出：
   ```bash
   (parted) quit
   ```

**后续操作（分区后必须做）**

1. **格式化分区**（创建文件系统）：

   根据需求，可以选择不同的文件系统

   ```bash
   mkfs.ext4 /dev/sdb1   # 假设分区是 sdb1
   # 或 mkfs.xfs /dev/sdb1
   ```

2. **挂载使用**：
   ```bash
   mkdir /mnt/mydisk
   mount /dev/sdb1 /mnt/mydisk
   ```

3. **设置开机自动挂载**（可选）：
   编辑 `/etc/fstab`，添加一行：

   ```
   /dev/sdb1  /mnt/mydisk  ext4  defaults  0 0
   ```

| 字段序号 | 字段意义           | 说明                                                         |
| -------- | ------------------ | ------------------------------------------------------------ |
| 1        | 文件系统           | 可以填写设备文件名、卷标或者 UUID                            |
| 2        | 挂载点             | 已存在的目录、绝对路径；特殊的分区（如 swap）不需要挂载点    |
| 3        | 文件系统类型       | 如 ext3、ext4                                                |
| 4        | 文件系统参数       | 多种参数组合，不同参数之间用 `" , "` 隔开（注意：引号内为逗号+空格） |
| 5        | 是否采用 dump 备份 | `0` = 不备份，`1` = 备份                                     |
| 6        | 开机是否自检       | `0` = 不采用 fsck 自检；`1` = 采用 fsck 自检                 |

**重要提醒**

- **备份数据**：分区会清除磁盘上所有原有数据！
- **确认设备名**：操作前反复核对 `lsblk` 输出，避免误删系统盘（如 `/dev/sda`）。
- **GPT vs MBR**：
  - 磁盘 >2TB → 必须用 GPT（`parted` + `mklabel gpt`）
  - 旧系统（如 Windows XP）可能不支持 GPT

完成以上步骤，你就成功创建并使用了新的磁盘分区！

### 2. 管理磁盘分区

在 Linux 系统中，`df` 和 `du` 是两个用于查看磁盘空间使用情况的核心命令，它们功能互补但原理不同。以下是详细介绍：

#### **1. `df` 命令（disk free）**

**作用**：显示**文件系统级别**的磁盘空间使用情况（如总容量、已用空间、可用空间、挂载点等）。
**特点**：

- 直接从文件系统的超级块（superblock）读取数据，**速度快**。
- 显示的是整个分区或挂载点的全局信息。
- 能统计**已被删除但仍在被进程占用**的文件（这些文件仍占用磁盘空间）。

#### **常用参数**：

```bash
df -h    # 以易读单位（GB/MB）显示
df -T    # 显示文件系统类型（如 ext4、xfs）
df /path # 查看指定路径所在分区的空间使用情况
```

#### **输出示例**：

```plaintext
文件系统        类型   容量  已用  可用 已用% 挂载点
/dev/sda1      ext4    50G   20G   30G   40%  /
```

#### **2. `du` 命令（disk usage）**

**作用**：计算**指定文件或目录**实际占用的磁盘空间。
**特点**：

- 通过遍历目录树逐个统计文件大小，**速度较慢**（尤其在大目录中）。
- 只统计当前存在的文件（**不包含已删除但未释放的文件**）。
- 默认显示目录下所有子目录和文件的大小。

#### **常用参数**：

```bash
du -h /path     # 以易读单位显示指定路径的大小
du -sh /path    # 仅显示总大小（-s 表示 summary）
du --max-depth=1 /path  # 只显示一级子目录的大小
du -a /path     # 递归显示
```

#### **输出示例**：

```plaintext
4.0K    ./file1.txt
8.0M    ./logs/
12M     .          # 当前目录总大小
```

| **对比项**     | **`df`**                 | **`du`**                 |
| -------------- | ------------------------ | ------------------------ |
| **统计对象**   | 整个文件系统（分区）     | 指定文件或目录           |
| **数据来源**   | 文件系统元数据（超级块） | 遍历文件系统中的实际文件 |
| **速度**       | 极快                     | 较慢（需递归扫描）       |
| **已删除文件** | 会计入（若被进程占用）   | 不计入                   |
| **典型用途**   | 查看磁盘剩余空间         | 定位大文件/目录          |

#### **4. 常见问题场景**

**场景 1：`df` 和 `du` 结果不一致**

- **原因**：有文件被删除但仍在被进程占用（如日志文件被 `rm` 但服务未重启）。
- **解决**：用 `lsof +L1` 找出被删除但仍打开的文件，重启相关进程。

**场景 2：快速定位大目录**

```bash
# 查看根目录下各子目录大小（人类可读）
du -h --max-depth=1 /
```

**场景 3：检查特定分区使用率**

```bash
# 查看 /home 分区的使用情况
df -h /home
```

总结

- 用 `df` **监控整体磁盘空间**（是否快满了？）。
- 用 `du` **分析具体文件/目录占用**（哪个文件占空间？）。
  两者结合可高效管理磁盘资源！

#### 5. `lsof`命令（List Open Files）

`lsof`（**List Open Files**）是 Linux 系统中一个极其强大的诊断和分析工具。由于 Linux 遵循“一切皆文件”的设计哲学，因此 `lsof` 不仅能列出普通文件，还能显示目录、网络连接、管道、设备、套接字等所有被进程打开的资源。

- **查看进程打开的文件**：包括普通文件、日志、库文件等。
- **监控网络连接**：识别哪个进程在使用特定端口或 IP。
- **排查文件占用问题**：例如无法删除文件（因被进程占用）。
- **安全审计**：发现异常进程或隐藏的网络连接。

#### 二、常用命令示例

1. **查看端口占用**

```bash
# 查看占用 8080 端口的进程
sudo lsof -i :8080

# 查看所有 TCP 连接
sudo lsof -i tcp

# 查看所有 UDP 连接
sudo lsof -i udp
```

2. **查看进程打开的文件**

```bash
# 查看 PID 为 1234 的进程打开了哪些文件
sudo lsof -p 1234

# 查看进程名对应的文件（如 nginx）
sudo lsof -c nginx
```

3. **查看用户打开的文件**

```bash
# 查看 root 用户打开的所有文件
sudo lsof -u root
```

4. **查看被删除但仍被占用的文件**

```bash
# 列出已被删除但未释放的文件（常用于排查磁盘空间不释放问题）
sudo lsof +L1
```

5. **查看指定目录/文件的占用情况**

```bash
# 查看 /var/log 目录被哪些进程使用
sudo lsof +D /var/log

# 查看某个具体文件（如 access.log）的占用进程
sudo lsof /var/log/access.log
```

#### 三、典型应用场景

**场景 1：端口冲突**

当启动服务时提示“Address already in use”，用以下命令快速定位：
```bash
sudo lsof -i :端口号
```

**场景 2：无法卸载磁盘**

如果 `umount` 提示“target is busy”，用以下命令找出占用进程：
```bash
sudo lsof /挂载点路径
```

**场景 3：磁盘空间未释放**

删除大文件后磁盘空间未释放？可能是文件被进程占用：
```bash
# 找出被删除但仍打开的文件
sudo lsof +L1
# 重启对应进程即可释放空间
```

**场景 4：安全排查**

检查是否有异常外连：
```bash
# 列出所有网络连接及对应进程
sudo lsof -i
```

#### 四、注意事项

1. **权限要求**： 
   普通用户只能查看自己的进程，需 `sudo` 查看全系统信息。
2. **输出字段说明**：  
   - `COMMAND`：进程名（是什么进程打开了这个文件）
   - `PID`：进程 ID  
   - `USER`：所属用户  
   - `FD`：文件描述符（如 `txt`=程序文件, `mem`=内存映射）  
   - `TYPE`：文件类型（如 `REG`=普通文件, `DIR`=目录）  
   - `NAME`：文件或网络地址
3. **性能影响**： 
   在大型系统中运行 `lsof` 可能短暂消耗较多资源（因其需遍历所有进程）。

#### 五、安装方法

多数 Linux 发行版默认未安装 `lsof`，可通过以下命令安装：
```bash
# CentOS/RHEL
sudo yum install -y lsof

# Ubuntu/Debian
sudo apt-get install -y lsof
```
