## 文件和目录管理

Linux 的文件和目录管理是系统操作的基础，涉及**组织、访问、修改和保护**数据。其核心建立在“多用户安全模型”和“一切皆文件”的哲学之上。

### 一、核心理念

#### 1. 一切皆文件（Everything is a file）

在 Linux 中，不仅普通文档是文件，以下也均以文件形式抽象：

- **普通文件**：文本、图片、脚本、二进制程序等
- **目录**：特殊文件，用于组织其他文件
- **设备文件**：如 `/dev/sda`（硬盘）、`/dev/tty`（终端）
- **虚拟文件**：如 `/proc/cpuinfo`（内核信息）、管道、套接字等

> 💡 所有路径都从根目录 `/` 开始。

#### 2. 多用户与权限隔离

Linux 是典型的多用户系统，不同用户拥有不同权限。为保障安全，系统对**所有者（user）**、**所属组（group）** 和 **其他人（others）** 访问同一文件的权限做了严格区分。

### 二、标准目录结构（FHS 规范）

```
/
├── /bin        → 必要的用户命令（如 ls, cp, mv）
├── /etc        → 系统配置文件（如 passwd, nginx.conf）
├── /home       → 普通用户家目录（如 /home/yubao）
├── /var        → 可变数据（日志 /var/log、邮件、数据库）
├── /tmp        → 临时文件（通常重启后清空）
├── /usr        → 用户级程序与只读数据（/usr/bin, /usr/share）
├── /proc       → 虚拟文件系统，反映内核与进程状态
└── /dev        → 设备文件（硬件接口抽象）
```

### 三、查看文件属性：`ls -l`

使用 `ls -l`（或 `ll`）可查看文件详细信息：

```bash
$ ls -l report.pdf
-rw-r----- 1 yubao finance 2048 Apr 5 10:00 report.pdf
```

#### 🔍 权限字段解析（共 10 个字符）

| 位置      | 含义                                                         |
| --------- | ------------------------------------------------------------ |
| 第 0 位   | **文件类型**：`-`=普通文件，`d`=目录，`l`=符号链接，`b`=块设备，`c`=字符设备 |
| 第 1–3 位 | **所有者（user）权限**                                       |
| 第 4–6 位 | **所属组（group）权限**                                      |
| 第 7–9 位 | **其他人（others）权限**                                     |

每组权限由 `r`（读）、`w`（写）、`x`（执行）组成，无权限用 `-` 表示。

> ✅ 示例：`-rw-r-----`  
>
> - 所有者：`rw-` → 可读写  
> - 组：`r--` → 只读  
> - 其他人：`---` → 无任何权限

> ⚠️ **注意**：  
>
> - 对**目录**而言，`r`=可列出文件列表，`w`=可增删文件，`x`=可进入（`cd`）  
> - 若目录无 `x` 权限，即使有 `r` 也无法访问内容！

### 四、常用文件与目录操作命令

| 功能         | 命令    | 常用选项                                   | 示例                 |
| ------------ | ------- | ------------------------------------------ | -------------------- |
| 列出内容     | `ls`    | `-l`（详细）`-a`（含隐藏）`-h`（易读大小） | `ls -lah /etc`       |
| 切换目录     | `cd`    | `..`（上一级）`~`（家目录）                | `cd ~/projects`      |
| 显示当前路径 | `pwd`   | —                                          | `pwd`                |
| 创建目录     | `mkdir` | `-p`（递归创建）                           | `mkdir -p a/b/c`     |
| 创建空文件   | `touch` | —                                          | `touch note.txt`     |
| 复制         | `cp`    | `-r`（递归目录）`-i`（覆盖确认）           | `cp -r src/ dest/`   |
| 移动/重命名  | `mv`    | `-i`（安全模式）                           | `mv old.txt new.txt` |
| 删除文件     | `rm`    | `-r`（删目录）`-f`（强制）                 | `rm -rf tmp/` ⚠️      |
| 删除空目录   | `rmdir` | —                                          | `rmdir empty_dir`    |

> ❗ `rm -rf /` 会删除整个系统！切勿在生产环境随意使用 `rm -rf`。

### 五、查找文件与内容

#### 1. `find` —— 按条件查找文件/目录（功能强大）

```bash
# 按名称查找（支持通配符）
find /home -name "*.log"

# 按类型查找（f=文件, d=目录, l=链接）
find /etc -type f -name "hosts"

# 按时间查找（最近7天修改过）
find /var/log -mtime -7

# 按大小查找（大于100MB）
find / -size +100M

# 查找并执行操作（谨慎！）
find . -name "*.tmp" -delete               # 删除所有 .tmp 文件
find . -name "*.sh" -exec chmod +x {} \;  # 给所有 .sh 文件加执行权限
```

> ✅ 优点：支持复杂条件组合（`-and`, `-or`, `-not`）
> ⚠️ 注意：默认递归搜索，大范围查找可能较慢

#### 2. `grep` —— 在文件中搜索文本内容（文本过滤神器）

```bash
# 基本搜索
grep "error" /var/log/syslog

# 忽略大小写
grep -i "warning" app.log

# 显示行号
grep -n "timeout" config.txt

# 递归搜索目录下所有文件
grep -r "password" /etc/

# 反向匹配（显示不包含关键词的行）
grep -v "^#" nginx.conf    # 过滤掉注释行（以#开头）

# 统计匹配行数
grep -c "failed" auth.log
```

> 💡 **经典组合**：`grep` 常与 `cat`、`tail`、`ps` 等命令配合使用（见管道部分）

#### 3. 管道（`|`）—— 命令链式处理（Linux 核心思想）

管道将**前一个命令的输出**作为**后一个命令的输入**，实现数据流处理。

##### 🌰 典型组合示例：

```bash
# 查看运行中的 nginx 进程
ps aux | grep nginx

# 查看非注释、非空行的配置
cat /etc/ssh/sshd_config | grep -v "^#" | grep -v "^$"

# 实时监控含 "ERROR" 的日志
tail -f /var/log/app.log | grep "ERROR"

# 统计当前目录下 .txt 文件数量
ls *.txt | wc -l

# 查找大日志文件并按大小排序
find /var/log -name "*.log" -size +10M -exec ls -lh {} \; | sort -k5 -hr
```

> ✅ 管道是 Linux “组合小工具完成大任务”哲学的体现
> 🔁 可串联多个命令：`cmd1 | cmd2 | cmd3 | ...`

### 六、查看文件内容

不同场景需选用不同命令，避免大文件刷屏或效率低下。

#### 1. `cat` —— 快速显示全文（适合小文件）

```bash
cat file.txt          # 显示全部内容
cat -n file.txt       # 显示行号
cat file1 file2       # 合并多个文件输出
```

> ✅ 优点：简单直接
> ❌ 缺点：大文件会快速滚屏，无法交互

#### 2. `less` —— 交互式分页查看（**推荐首选**）

```bash
less large.log
```

**常用操作**：

- `空格` / `f`：下一页
- `b`：上一页
- `/关键词`：向下搜索
- `?关键词`：向上搜索
- `g`：跳到开头
- `G`：跳到末尾
- `q`：退出

> ✅ 优点：支持前后翻页、搜索、大文件高效加载
> 💡 提示：比 `more` 更强大，是现代 Linux 默认推荐工具

#### 3. `more` —— 基础分页查看（单向滚动）

```bash
more config.conf
```

- `空格`：下一页
- `Enter`：下一行
- `q`：退出

> ⚠️ 缺点：**不能向上翻页**，功能较 `less` 弱

#### 4. `head` —— 查看文件开头（默认前 10 行）

```bash
head /var/log/syslog           # 前 10 行
head -n 20 error.log           # 前 20 行
head -c 100 file.bin           # 前 100 字节
```

#### 5. `tail` —— 查看文件末尾（默认后 10 行）

```bash
tail access.log                # 后 10 行
tail -n 50 debug.log           # 后 50 行
tail -f /var/log/nginx.log     # **实时跟踪日志更新**（运维必备！）
```

> 🔥 `tail -f` 是监控日志的核心命令，按 `Ctrl+C` 退出

#### 6. `nl` —— 带行号显示（比 `cat -n` 更规范）

```bash
nl script.sh    # 自动编号非空行（空行不编号）
```

#### 7. `tac` —— 反向显示（`cat` 倒过来）

```bash
tac history.txt   # 从最后一行开始向上显示
```

> 适用于查看日志时快速定位最新记录

#### 命令对比速查表

| 命令   | 适用场景                   | 是否分页 | 可否反向 | 实时跟踪           |
| ------ | -------------------------- | -------- | -------- | ------------------ |
| `cat`  | 小文件全文显示             | ❌        | ❌        | ❌                  |
| `less` | **通用首选**（尤其大文件） | ✅        | ✅        | ❌（但可 `F` 模拟） |
| `more` | 基础分页（老旧系统）       | ✅        | ❌        | ❌                  |
| `head` | 查看开头                   | ❌        | ❌        | ❌                  |
| `tail` | 查看结尾 / **实时日志**    | ❌        | ❌        | ✅（`-f`）          |
| `nl`   | 需要规范行号               | ❌        | ❌        | ❌                  |
| `tac`  | 反向查看（如倒序日志）     | ❌        | ✅        | ❌                  |

### 七、权限管理：`chmod` 与归属管理：`chown` / `chgrp`

#### 1. 修改文件归属

##### `chown`（change owner）— 修改所有者和/或所属组

```bash
# 仅改所有者
chown yubao file.txt

# 同时改所有者和组
chown yubao:dev report.pdf

# 递归修改整个目录
chown -R www-data:www-data /var/www
```

##### `chgrp`（change group）— 仅修改组（较少用，`chown` 更通用）

```bash
chgrp developers project/
```

#### 2. 修改访问权限：`chmod`

##### 方法一：数字模式（八进制）

权限值：

- `r = 4`
- `w = 2`
- `x = 1`

每类用户权限相加，形成三位数：

| 权限组合 | 数值 |
| -------- | ---- |
| `rwx`    | 7    |
| `rw-`    | 6    |
| `r-x`    | 5    |
| `r--`    | 4    |
| `---`    | 0    |

✅ 示例：

```bash
chmod 640 report.pdf    # u=rw, g=r, o=---
chmod 755 script.sh     # u=rwx, g=rx, o=rx
chmod 700 private/      # 仅所有者可访问（私密目录）
```

> 加 `-R` 可递归应用到目录及子内容。

##### 方法二：符号模式（更直观）

语法：`chmod [ugoa][+-=][rwx] 文件`

| 符号 | 含义              |
| ---- | ----------------- |
| `u`  | 所有者            |
| `g`  | 所属组            |
| `o`  | 其他人            |
| `a`  | 所有人（= u+g+o） |
| `+`  | 添加权限          |
| `-`  | 移除权限          |
| `=`  | 设定权限（覆盖）  |

✅ 示例：

```bash
# 设置为 rwxr-xr--
chmod u=rwx,g=rx,o=r file

# 给所有人加执行权限
chmod a+x run.sh

# 去掉所有人的执行权限
chmod a-x *.txt

# 给同组用户加写权限
chmod g+w shared.log
```

### 八、安全实践

| 场景                        | 推荐权限                                                |
| --------------------------- | ------------------------------------------------------- |
| 普通配置文件                | `644`（所有者可改，组和其他人只读）                     |
| 可执行脚本                  | `755`                                                   |
| 用户私密目录（如 `~/.ssh`） | `700`                                                   |
| Web 应用目录                | 目录 `755`，文件 `644`，敏感配置 `600`                  |
| 团队共享目录                | `770` + `setgid`（`chmod g+s dir`，使新文件自动继承组） |

> ❌ **绝对避免**：随意使用 `chmod 777` —— 这等于向全世界开放写权限，极易被利用！

### 九、总结

> **Linux 文件管理 = 路径导航 + 命令操作 + 权限控制 + 安全意识**

掌握以下核心能力，即可高效、安全地管理文件系统：

1. **理解权限模型**：user/group/others + r/w/x

2. 熟练使用基础命令

   ：

   - 操作：`ls`, `cp`, `mv`, `rm`, `mkdir`, `touch`
   - **查找：`find`（找文件），`grep`（找内容）**
   - **组合：管道 `|` 实现命令协同**
   - 查看：`cat`, `less`, `head`, `tail -f`
   - 权限：`chmod`, `chown`

3. **遵循最小权限原则**：只给必要的访问能力

> 💡 **终极建议**：
>
> - 日常查看 → 用 `less`
> - 监控日志 → 用 `tail -f`
> - 定位文件 → 用 `find`
> - 搜索内容 → 用 `grep`
> - 复杂任务 → 用 **管道 `|` 串联命令**
